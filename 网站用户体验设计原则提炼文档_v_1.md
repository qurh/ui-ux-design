# 网站用户体验设计原则提炼文档 v1.0

本文档基于多个优秀网站 UX 案例（Wise / Miro / Airbnb / Apple / Notion / Suno / Temu 等）抽象提炼可复用的用户体验设计原则、方法体系与策略模式，适用于：

- SaaS 产品设计
- AI 产品设计
- RAG 系统入口设计
- Agent 产品入口设计
- Web / App 产品信息架构设计
- 转化型网站设计

目标：形成可迁移、可复用、可工程化落地的 UX 设计策略框架。

---

# 一、UX 的核心本质模型（第一性原则）

优秀 UX ≠ 视觉设计优秀

优秀 UX = 降低理解成本 + 降低操作成本 + 降低决策成本

可以抽象为：

UX = Clarity + Flow + Trust + Speed

即：

清晰度 + 流动性 + 信任感 + 响应速度

这是所有优秀网站反复出现的底层结构。

---

# 二、首屏价值传达原则（First Screen Value Principle）

核心规则：

用户在滚动之前必须理解三件事：

1. 这是做什么的
2. 适合谁
3. 为什么值得继续看

设计方法：

推荐结构：

标题：产品核心价值
副标题：目标用户 + 使用场景
交互元素：产品预览 / demo / 示例
CTA：下一步探索入口

典型模式：

Suno
Wise
Notion
Apple

设计策略总结：

价值不是解释出来的
价值必须被体验到

---

# 三、产品即体验原则（Product-as-Experience Principle）

核心思想：

不要介绍产品
要让用户体验产品

实现方式：

常见模式：

首页直接可操作
首页模拟真实使用场景
首页提供轻量级交互
首页提供 demo 体验

典型案例：

Miro：首页即画布
Wise：首页即汇率计算器
Suno：首页即提示输入

适用于：

AI 产品
SaaS 产品
数据分析工具
Agent 产品

---

# 四、低摩擦路径原则（Low Friction Path Principle）

核心思想：

不要过早索取用户承诺

典型错误：

访问即注册
访问即登录
访问即填写表单

正确策略：

先体验
再理解
再信任
最后注册

推荐流程模型：

探索 → 体验 → 理解 → 信任 → 注册

典型案例：

Airbnb
Wise
Notion
Preply

适用于：

AI 产品
知识产品
教育产品
工具产品

---

# 五、认知负荷控制原则（Cognitive Load Control Principle）

核心思想：

一次只让用户做一个决策

实现方式：

减少同时出现的选择
减少视觉干扰
减少解释文本
减少字段数量
减少流程复杂度

典型实现方式：

渐进式信息披露
单主操作页面
步骤化流程设计

典型案例：

Apple
Airbnb
Preply

设计目标：

减少犹豫
减少错误
减少退出

---

# 六、意图驱动导航原则（Intent-driven Navigation Principle）

核心思想：

导航应该围绕用户意图
而不是系统结构

典型错误：

按照组织架构设计导航
按照功能模块设计导航

正确方式：

按照用户任务设计导航

典型用户意图示例：

我要了解价格
我要比较方案
我要看看怎么使用
我要立即开始

典型案例：

Cinépolis
Airbnb
Notion

适用于：

复杂系统
平台型产品
多角色系统
Agent 系统

---

# 七、渐进式披露原则（Progressive Disclosure Principle）

核心思想：

先展示必要信息
再展示详细信息

不要一次展示全部复杂度

实现方式：

第一层：核心价值
第二层：关键功能
第三层：高级能力
第四层：技术细节

典型案例：

Apple 产品比较页面
Notion 产品展示页面
Airbnb 房源展示流程

适用于：

复杂产品
技术产品
AI 产品
数据产品

---

# 八、信任前置原则（Trust-before-Commitment Principle）

核心思想：

信任必须先于注册

信任构建方式：

真实案例
用户评价
透明价格
实时反馈
产品预览
交互体验

典型案例：

Airbnb
Wise
GOAT
Apple

适用于：

金融产品
医疗产品
AI 产品
高风险决策产品

---

# 九、微交互引导原则（Microinteraction Guidance Principle）

核心思想：

不要用说明引导用户
用反馈引导用户

实现方式：

hover 动效
按钮反馈
动画提示
步骤提示
状态变化

典型作用：

减少犹豫
增强理解
增加掌控感

典型案例：

Miro
Notion
Apple

---

# 十、探索优先原则（Explore-first Strategy）

核心思想：

允许用户探索
而不是立即转化

典型设计方式：

允许浏览内容
允许比较内容
允许体验功能
允许查看价格

之后再要求注册

典型案例：

Airbnb
Notion
Wise

适用于：

SaaS
AI
教育产品
平台产品

---

# 十一、视觉即功能原则（Visual-as-Function Principle）

核心思想：

视觉不仅是装饰
视觉是理解工具

典型实现方式：

视觉比较
视觉反馈
视觉引导
视觉结构表达

典型案例：

Apple
Notion
Airbnb

适用于：

复杂产品
配置型产品
比较型产品

---

# 十二、行为动量设计原则（Momentum-driven UX Principle）

核心思想：

让用户持续前进
不要中断用户流程

实现方式：

步骤连续
反馈即时
进度可见
路径清晰

典型案例：

Preply onboarding
Apple 购买流程
Notion 学习路径

设计目标：

减少退出率
提高完成率
提高转化率

---

# 十三、身份感与社区感设计原则（Identity-driven UX Principle）

核心思想：

用户体验可以强化身份认同

实现方式：

稀缺性
认证机制
社区文化
等级结构
专属体验

典型案例：

GOAT
Airbnb 房东体系

适用于：

社区产品
交易平台
会员体系

---

# 十四、游戏化行为驱动原则（Gamification-driven UX Principle）

核心思想：

通过奖励机制影响行为

实现方式：

限时奖励
任务机制
等级机制
优惠机制
倒计时机制

典型案例：

Temu

适用于：

电商
增长系统
用户留存系统

注意：

适用于增长导向产品
不适用于信任导向产品

---

# 十五、一致性交互原则（Consistency Principle）

核心思想：

一致性降低学习成本

一致性包括：

布局一致
交互一致
视觉一致
语言一致
反馈一致

典型案例：

Apple
Notion
Airbnb

设计收益：

减少学习成本
提升熟练速度
增强控制感

---

# 十六、真实意图对齐原则（Intent Alignment Principle）

核心思想：

系统结构必须符合用户思维结构

设计方法：

用户访谈
行为分析
路径分析
可用性测试

典型用户意图结构：

比较
理解
尝试
决定
购买

这是 UX 信息架构设计的核心依据

试用

AI 产品官网
Agent 官网
知识产品官网
工具型产品官网

---

# 十八、优秀 UX 的统一设计目标模型

所有优秀 UX 本质统一目标：

减少三种成本：

理解成本
操作成本
决策成本

增加三种能力：

信任感
掌控感
确定性

这构成 UX 设计的统一评价标准。
